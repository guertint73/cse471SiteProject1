Component Details

This component implements an effects module. In each instrumentâ€™s XML tag, an effect for that instrument is specified by name. The effect is converted to a number and is stored as a member variable in the CInstrument object. This is then processed when the frames are created in CSynthesizer. The possible effects for instruments are Chorus, Flange, and Reverberation. Each of these effects are classes in the code, and are defined as objects in CSynthesizer. They each use the function Process(frameIn, frameOut, time) to perform the effect processing.


Chorus and Flange both calculate a DELAY value based on the current time of the score. The Chorus delay amount varies from 20 to 30 milliseconds at a sinusoidal rate of .25Hz. The Flange delay amount varies from 2 to 10 milliseconds at a sinusoidal rate of .25Hz. An input queue is used to store the input values needed to achieve these effects. Reverb has a set delay length of .238 milliseconds. The delayed output frame is multiplied by .5 then added to the current input frame, making the sound echo and fade over time. An output queue is used to store the output values needed to achieve this effect.


NoiseGate samples the previous 100 frames and checks if any of them are above the threshold value. If this is found to be true, the sound is allowed through the noise gate. If this is false, the sound is reduced to 0 and nothing passes through the gate. A queue was used to store the samples that were compared to the threshold.

Grading Criteria Achieved

The grading elements supported are:

  1. 1. Waveform playback from tables
  2. 2. Varying pitch playback from tables
  3. 3. Envelope Generation
  4. 4. Polyphony
  5. 5. Reson Implementation
  6. 6. Filter Envelopes (sweeping the Moog sound)
